Project Overview

Research Question

How does stay duration impact mental health outcomes among international student populations? This project analyses three validated psychological scales — PHQ-9, SCS, and ASISS — across student cohorts segmented by length of stay, using enterprise SQL patterns to surface actionable insights.

Query Highlights

A selection of 25+ advanced SQL queries written for this project, demonstrating enterprise-level analytical capability.

student-stay-duration-score-analysis.sql SQL 
-- Core analytical query: stay duration impact on all three wellbeing metrics
-- Returns 9 rows × 5 columns for international students only

SELECT
    stay,
    COUNT(*) AS count_int,
    ROUND(AVG(todep), 2)  AS average_phq,
    ROUND(AVG(tosc),  2)  AS average_scs,
    ROUND(AVG(toas),  2)  AS average_as
FROM   students
WHERE  inter_dom = 'Inter'
GROUP  BY stay
ORDER  BY stay DESC;
advanced-student-wellbeing-sql-analysis.sql — Query 3 SQL 
-- Advanced stay duration analysis using window functions
-- LAG/LEAD track trends; RANK identifies highest-risk cohorts

WITH duration_base AS (
    SELECT
        stay,
        COUNT(*)                      AS student_count,
        ROUND(AVG(todep), 2)         AS avg_depression,
        ROUND(AVG(tosc),  2)         AS avg_social,
        ROUND(AVG(toas),  2)         AS avg_anxiety,
        ROUND(STDDEV(todep), 2)      AS depression_volatility
    FROM students
    WHERE inter_dom = 'Inter'
    GROUP BY stay
),
duration_trends AS (
    SELECT
        *,
        -- Lag/Lead for period-over-period trend
        LAG(avg_depression, 1)  OVER (ORDER BY stay) AS prev_depression,
        LEAD(avg_depression, 1) OVER (ORDER BY stay) AS next_depression,
        LAG(avg_anxiety, 1)     OVER (ORDER BY stay) AS prev_anxiety,

        -- Ranking: which cohort has highest risk?
        RANK() OVER (ORDER BY avg_depression DESC) AS depression_risk_rank,
        RANK() OVER (ORDER BY avg_anxiety    DESC) AS anxiety_risk_rank,
        RANK() OVER (ORDER BY avg_social     ASC)  AS isolation_rank
    FROM duration_base
)
SELECT * FROM duration_trends
ORDER BY stay;
advanced-student-wellbeing-sql-analysis.sql — Query 4 SQL 
-- Multi-dimensional risk scoring algorithm using Z-scores and CTEs
-- Identifies at-risk students with statistical confidence

WITH population_benchmarks AS (
    SELECT
        ROUND(AVG(todep),    2) AS pop_avg_depression,
        ROUND(STDDEV(todep), 2) AS pop_std_depression,
        ROUND(AVG(tosc),     2) AS pop_avg_social,
        ROUND(STDDEV(tosc),  2) AS pop_std_social,
        ROUND(AVG(toas),     2) AS pop_avg_anxiety,
        ROUND(STDDEV(toas),  2) AS pop_std_anxiety
    FROM students
),
individual_z_scores AS (
    SELECT
        s.*,
        -- Z-score = (value − mean) / stddev
        ROUND((s.todep - pb.pop_avg_depression)
              / NULLIF(pb.pop_std_depression, 0), 3) AS depression_z,
        ROUND((s.tosc  - pb.pop_avg_social)
              / NULLIF(pb.pop_std_social,      0), 3) AS social_z,
        ROUND((s.toas  - pb.pop_avg_anxiety)
              / NULLIF(pb.pop_std_anxiety,     0), 3) AS anxiety_z
    FROM students s
    CROSS JOIN population_benchmarks pb
),
risk_classified AS (
    SELECT
        *,
        CASE
            WHEN depression_z > 1.5 AND anxiety_z > 1.5 THEN 'CRITICAL'
            WHEN depression_z > 1.0 OR  anxiety_z > 1.0 THEN 'HIGH'
            WHEN social_z     < -1.0                      THEN 'MODERATE'
            ELSE                                                   'LOW'
        END AS risk_tier
    FROM individual_z_scores
)
SELECT risk_tier, COUNT(*) AS student_count
FROM   risk_classified
GROUP  BY risk_tier
ORDER  BY student_count DESC;
advanced-student-wellbeing-sql-analysis.sql — ETL SQL 
-- Star Schema: Fact table ETL with complex transformations
-- Populates fact_student_wellbeing from normalised source

INSERT INTO fact_student_wellbeing (
    student_key, time_key, region_key,
    depression_score, social_score, anxiety_score,
    risk_tier, stay_category
)
SELECT
    ds.student_key,
    dt.time_key,
    dr.region_key,

    -- Source metrics
    s.todep  AS depression_score,
    s.tosc   AS social_score,
    s.toas   AS anxiety_score,

    -- Derived risk classification
    CASE
        WHEN s.todep >= 15 AND s.toas >= 80 THEN 'CRITICAL'
        WHEN s.todep >= 10 OR  s.toas >= 70 THEN 'HIGH'
        WHEN s.tosc  <= 30                   THEN 'MODERATE'
        ELSE                                       'LOW'
    END  AS risk_tier,

    -- Stay duration bucketing
    CASE
        WHEN s.stay BETWEEN 1 AND  2 THEN 'Short'
        WHEN s.stay BETWEEN 3 AND  5 THEN 'Medium'
        ELSE                             'Long'
    END  AS stay_category

FROM       students         s
JOIN  dim_student    ds  ON ds.source_id  = s.id
JOIN  dim_time       dt  ON dt.survey_year = 2019
JOIN  dim_region     dr  ON dr.region_code = s.region
WHERE s.inter_dom = 'Inter';

Data Architecture

End-to-End Pipeline

From raw CSV to a fully normalised, star-schema data warehouse with OLAP-ready analytical layers.

STEP 1

Raw Ingestion

286-row CSV imported with 50 columns covering demographics, language proficiency, mental health scores, and support system usage.

STEP 2

Normalisation

Progressive design from denormalised flat file through 1NF, 2NF, and 3NF — eliminating transitive dependencies and redundancy.

STEP 3

Star Schema

Dimensional model with fact_student_wellbeing at the centre, surrounded by region, time, student, and advisor dimensions.

STEP 4

ETL Pipeline

Automated population of dimensions and fact tables including SCD Type 2 tracking, time hierarchy generation, and risk classification.

STEP 5

OLAP Analytics

Multi-dimensional drill-down, slice/dice, pivot analysis and risk scoring dashboards powered by the warehouse layer.
